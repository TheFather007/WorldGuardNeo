## Требования

| Что | Версия |
| --- | --- |
| JDK | **21** (обязательно) |
| NeoForge | 21.1.233 (диапазон `[21.1.0,)`) |
| Minecraft | 1.21.1 |
| Gradle | обёртка в комплекте (`./gradlew`) — ставить отдельно не нужно |

Среда сборки настроена под NeoForge 1.21.1 и Java 21. Gradle при первом запуске сам скачает нужный JDK, если его нет, но какая-то версия Java для старта обёртки всё равно требуется.

## Быстрая сборка

```bash
./gradlew clean build
```

Готовый `.jar` появится в `build/libs/`. Это и есть файл для папки `mods/` сервера.

## Структура проекта

Структура соответствует эталону NeoForge MDK / mod-generator:

```
worldguardneo/
├── build.gradle.kts          # конфигурация сборки (server-only)
├── gradle.properties         # версии, метаданные мода
├── settings.gradle.kts
├── gradlew / gradlew.bat     # обёртка Gradle
└── src/main/
    ├── java/                 # исходный код (60 файлов)
    └── resources/
        ├── META-INF/neoforge.mods.toml
        ├── assets/worldguardneo/lang/   # en_us, ru_ru
        ├── worldguardneo.mixins.json
        └── pack.mcmeta
```

## Зависимости

* **WorldEdit** (≥ 7.3) — **обязательная зависимость в рантайме**. Регионы создаются исключительно по выделению WorldEdit (`//wand`, `//pos1`/`//pos2`, `//sel poly`). Без WorldEdit мод не загрузится. **Для компиляции WorldEdit не нужен** — мод обращается к нему только через рефлексию, поэтому собирается без WorldEdit в classpath.
* **LuckPerms** (≥ 5.4) — опционально. Тонкая настройка прав через группы и контексты + лимиты регионов на группу. Без LuckPerms все права маппятся на op-уровни.
* **BlueMap** / **squaremap** — опционально. Отображение регионов на веб-карте.
* **WorldEditCUI** (только клиент) — визуальная подсветка выделений. Её обеспечивает сам WorldEdit; ставить ничего на сервер не нужно.

Для альтернативных хранилищ регионов (`storage-format`) нужен соответствующий JDBC-драйвер в classpath сервера: **sqlite-jdbc** для `sqlite`, **H2** для `h2` (его уже поставляет LuckPerms), **mysql-connector-j** *или* драйвер **MariaDB** для `mysql`. Драйверы ищутся по всем class-loader'ам и используются напрямую (минуя `DriverManager`), поэтому драйвер, загруженный другим модом, тоже работает. Без драйвера мод автоматически откатывается на `json`.

## Сборка без зависимостей компиляции

Мод намеренно спроектирован так, чтобы компилироваться без сторонних модов в classpath:

- **WorldEdit** — только рефлексия (`Class.forName`), прямых импортов нет.
- **LuckPerms**, **night-config (TOML)**, **fastutil**, **Mixin** — объявлены как `compileOnly` и предоставляются в рантайме самим NeoForge или установленными модами.
- **JDBC-драйверы** (sqlite/H2/MySQL) — загружаются рефлективно в рантайме, в сборке не участвуют.

Поэтому `./gradlew build` проходит «из коробки», без ручной установки чего-либо.

## Установка собранного мода

1. Положите `worldguardneo-1.2.jar` в каталог `mods/` сервера.
2. *(Опционально)* добавьте **WorldEdit** (нужен только для создания регионов через `/rg claim` и `/rg redefine`), LuckPerms, BlueMap, squaremap или JDBC-драйвер.
3. Запустите сервер один раз — будет создан `config/worldguardneo/config.toml`.
4. При необходимости отредактируйте конфиг и выполните `/rg reload`.

## Решение проблем

**`Could not find com.sk89q.worldedit:worldedit-neoforge:...`** — этой ошибки в текущей версии быть не должно: WorldEdit убран из зависимостей сборки. Если она появилась, убедитесь, что используете актуальный `build.gradle.kts` (без строки `compileOnly("com.sk89q.worldedit...")`).

**Gradle не находит JDK 21** — установите JDK 21 или дайте Gradle скачать его автоматически (нужна любая Java для старта обёртки).
