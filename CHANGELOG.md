## build 18 (v1.3) — 2026-06-26

Сборка, производительность, новые команды/события и расширенное покрытие тестами.

**Платформа.** Сборка переведена на **Gradle 9.6.0** + ModDevGradle 2.0.141. **NeoForge 21.1.234**
(диапазон `[21.1.0,)`). CI прогоняет батарею геймтестов по матрице версий **21.1.209 / 21.1.221 /
21.1.234**; пол матрицы — 21.1.209 (тестовый стенд использует более новый конструктор события).

**Производительность (для многолюдных серверов).** Асинхронная запись хранилища (write-behind):
снапшот сериализуется в серверном потоке, блокирующий I/O выполняет отдельный упорядоченный воркер —
MySQL-round-trip больше не стопорит тик. Пер-чанк кеш кандидатов в spatial-index. `onTreeGrow` —
один area-запрос вместо 48. Версионирование схемы хранилища (`<table>_meta`).

**Новые команды.** `/rg rename <id> <newId>` — переименование с сохранением данных (узел
`region.rename`, OP 2). `/rg audit <id> [limit]` — последние изменения региона из аудит-лога (узел
`region.audit`, OP 2). `/rg flags <id>` — кликабельный редактор флагов (кнопки allow/deny/clear/set).
Операнд `-w <world>` для id-команд (info/remove/flag/priority/setparent/owners/members). `/rg debug`
теперь показывает бэкенд+схему хранилища, глубину очереди записи и hit/miss кеша. **`/rg teleport`
теперь требует OP 2** (ранее OP 0) — либо узел `region.teleport`.

**Флаги и метаданные.** Новые флаги взаимодействия с существами: `villager-trade`, `ride`,
`entity-leash` (учитывают модель участников). Метаданные региона `created/modified/createdBy` в
`/rg info`. Кламп Y у флагов `spawn`/`teleport` к высоте мира (защита от выброса в пустоту).

**Конфиг и API.** Версионируемый `config.toml` с миграцией ключей. Событие `RegionFlagChangeEvent`
(изменение флага: флаг, старое/новое значение, группа, актор).

**Локализация.** Добавлены немецкий, испанский, французский, упрощённый китайский (всего 6 языков).

**Качество.** Аудит-проход с исправлениями (групповой фильтр флага не сбрасывается при правке
значения; `transfer` обновляет `modifiedAt`; ресинк инвентаря при отмене поводка/торговли; и др.).
Геймтест-батарея пересмотрена: убран дублирующий per-flag генератор, добавлены реальные тесты
через настоящий событийный/тик-путь — **166 enforcement-тестов** + пер-региональный round-trip
хранилища. Документация актуализирована.

## build 17 (v1.3) — 2026-06-17

Расширение возможностей поверх встроенного выделения: новые команды, флаги защиты, операционные
инструменты и интеграции.

**Команды выделения/регионов.** `/rg pos1|pos2 <x y z>` — явные координаты (поддержка `~` и консоли
через `/execute as`, узел `selection.pos.coords`, OP 4). `/rg select <id>` — загрузка геометрии
региона в выделение (узел `region.select`, OP 2). `/rg transfer <id> <player>` — передача
единоличного владения (узел `region.transfer`, OP 3). Палочка показывает live-размеры выделения
(Ш×В×Д + объём / число точек) в actionbar при кликах.

**Новые флаги защиты** (все по умолчанию allow): `vehicle-place`, `vehicle-enter`,
`item-frame-rotate`, `sign-edit`, `lectern-take`, `armor-stand-use`, `glide` (контроль элитр),
`bucket-fill`/`bucket-empty`, а также `spawn-limit` — лимиты спавна по типу (`entity-id:max`).

**Операционная надёжность.** Аудит-лог (`logs/worldguardneo-audit.log`) — кто/что/когда по
create/redefine/remove/undo/transfer/flag/priority/parent/членству. Soft-delete: `/rg remove`
кладёт регион в посессионную корзину, `/rg undo` восстанавливает (узел `region.undo`, OP 2).
`/rg migrate <json|sqlite|h2|mysql>` — конвертация данных между бэкендами (узел `migrate`, OP 4;
активируется после перезапуска).

**Интеграции/скриптинг.** Маркеры регионов на BlueMap и squaremap получили всплывающее окно по
клику (приоритет, владельцы, участники, флаги). KubeJS: публичный фасад `WorldGuardNeoAPI`
вызывается из скриптов через `Java.loadClass`; добавлены удобные методы (`getRegionAt` по int,
`queryFlag`/`queryValue` по имени флага) и руководство [KUBEJS.md](KUBEJS.md).

## build 16 (v1.3) — 2026-06-17

Крупное расширение: полностью свой инструмент выделения вместо WorldEdit, истечение неактивных
клеймов, командные флаги входа/выхода, инкрементальное хранилище и пара исправлений
хранилища/производительности.

**Полный отказ от WorldEdit.** Зависимость от WorldEdit (в т.ч. рефлективный мост) убрана. Выделение
регионов теперь встроено: команда `/rg wand` выдаёт предмет-палочку (по умолчанию `minecraft:stick`,
меняется через `selection.wand-item`). ЛКМ — точка 1, ПКМ — точка 2 для кубоида; режим полигона
(`/rg sel poly`) набирает вершины кликами. Также есть `/rg pos1`, `/rg pos2`, `/rg point`,
`/rg sel <cuboid|poly|clear>`. Защита предмета: выдаётся **только один раз** (если палочка уже в
инвентаре — повторно не даётся) и используется **только для выделения** (обычное действие предмета
гасится). Контур выделения и `/rg info` рисуются на клиенте через протокол `worldedit:cui`
(плагин-канал WorldEditCUI) — серверный мод сам шлёт CUI-сообщения, WorldEdit не нужен. У каждой
команды выделения свой узел прав (все OP 0): `worldguardneo.selection.wand` (`/rg wand`), `.mode`
(`/rg sel`), `.pos1`, `.pos2`, `.point`; `selection.use` теперь гейтит саму палочку (клики).

**Истечение клеймов** (по умолчанию выключено, `claim-expiry.enabled`). Авто-удаление регионов,
все владельцы которых офлайн дольше `claim-expiry.days`. Онлайн-владелец всегда «активен»;
безвладельные/админ-регионы не трогаются; never-seen-владелец считается активным. Скан при старте
и каждые `check-hours`, плюс вручную `/rg cleanup`. Активность — `activity.json`.

**Командные флаги входа/выхода** `on-entry` / `on-exit` — выполняют консольную команду при
пересечении границы (`%player%`, `%region%`, `%world%`), задаются только админами. (`allowed-cmds`
/`blocked-cmds` уже существовали.)

**Флаг `receive-chat` теперь работает** — реализован через миксин на доставке чата получателю;
игрок в регионе с `receive-chat=deny` не получает чужой чат. Неприменимый в NeoForge 1.21.1 флаг
`allowed-enchants` удалён (нет события, несущего и зачарование, и позицию; загрузка старых данных
не ломается — неизвестные флаги пропускаются).

**Инкрементальное хранилище** для SQLite/H2/MySQL — схема «строка на регион»
(`PRIMARY KEY(world, region_id)`): правка одного флага апдейтит одну строку, а не переписывает весь
мир. Каскадные операции (remove/redefine) по-прежнему делают полную синхронизацию. Ленивая
по-мировая миграция со старой blob-таблицы (сохраняется как бэкап). JSON-бэкенд без изменений.

**Хранилище/производительность:** WAL-checkpoint (SQLite) и `CHECKPOINT SYNC` (H2) перед файловым
бэкапом — снимок больше не теряет последние коммиты. Аллокационно-free проба границ для
`NeighborNotifyEvent` (вода/лава/огонь) — самый частый блок-ивент теперь без аллокаций.

Версия мода — **1.3**. Сборка зелёная.

## build 15 (v1.2) — 2026-06-14

Большой сквозной аудит кода с исправлением реальных дефектов корректности, безопасности и
потери данных, плюс точечная оптимизация горячих путей. Версия мода поднята до **1.2**.

**Потеря данных (хранилище):**
- Битый/обрезанный файл регионов больше не приводит к необратимой потере: повреждённые данные
  уходят в карантин (`<имя>.corrupt-<метка>` для JSON, отдельная строка для SQLite/H2/MySQL) до
  опустошения мира, поэтому следующее автосохранение их не затирает. Раньше защита от этого была
  только в одном месте.
- JSON-запись теперь fsync'ится перед атомарным переименованием (потеря питания не оставит
  обрезанный файл); кодировка явно UTF-8; в MySQL-таблице задан `utf8mb4`.

**Корректность защиты:**
- `frosted-ice-melt` **снова работает**: миксин инжектил в `randomTick`, а Frosted Ice тает через
  плановый `tick` — из-за `require=0` инжект молча не применялся, и флаг не действовал вообще.
- Мировой выключатель `useRegions=false` теперь уважают взрывы и цепочка смерти (keep-inventory/
  keep-xp/spawn) — раньше эти флаги срабатывали даже в мире с отключённым WGN.
- Полигональные регионы: крайний ряд блоков (+X/+Z) теперь защищён от игроков (ray-cast округляет
  координаты до блока, как у кубоида и WorldEdit).
- `exp-drops` (флаг для мобов) больше не уничтожает опыт игрока при смерти; доступ к контейнерам
  fail-safe при ошибке определения блока; регион на краю мира не выпадает из пространственного
  индекса.

**Поведение игрока:**
- Флаг `game-mode` теперь восстанавливает прежний режим при выходе из региона (раньше навсегда
  запирал игрока в наложенном режиме).
- Флаг `max-speed` применяется сессионным (transient) модификатором атрибута: выход из игры внутри
  такого региона больше не «впекает» скорость в сейв; подход без снимков и самокорректирующийся.

**Публичный API:**
- `RegionFlagDeniedEvent` (хук переопределения отказа) теперь порождается и при установке блока, не
  только при ломании; документация события исправлена (поля region/flag/actor/reason, без позиции).

**Оптимизация горячих путей** (код уже был сильно оптимизирован — только безопасные точечные правки):
переиспользование списка регионов в `onLivingDamage` и `denyMessage`, ленивое построение
notify-сообщения.

Прочие правки: устаревшая документация после перехода WorldEdit в опциональные приведена в
соответствие (`WorldEditAdapter`, `OpResolver`). Аудит подтвердил высокое качество остального кода
(потокобезопасность, парсеры, права, геометрия, бэкапы, локализация — без замечаний); сборка зелёная.

## build 14 — 2026-06-14

Критическое исправление видимости команд и переход WorldEdit в опциональные зависимости —
плюс чистка репозитория до «только мод».

**Исправлена видимость `/rg` и `/region`** (`LuckPermsResolver.has`). Раньше при результате
`UNDEFINED` (узел не упомянут в LuckPerms — обычный случай, когда правами управляют через
группы) метод жёстко возвращал deny и **игнорировал уровень OP**. Brigadier удаляет из дерева
команд игрока любую ветку, чей `requires` ложный, поэтому всё дерево `/rg` пропадало —
команды выглядели как «несуществующие». Теперь модель стандартная «LuckPerms переопределяет,
OP — это дефолт»: `TRUE` → allow, `FALSE` → deny (LP может отозвать доступ у OP),
`UNDEFINED` → откат на OP-уровень узла (`OpResolver`). `region.bypass` остаётся OP-5 и по-прежнему
никогда не выдаётся одним лишь OP.

**WorldEdit теперь опционален** (`neoforge.mods.toml`: `type="required"` → `type="optional"`).
Раньше отсутствие WorldEdit заставляло NeoForge вообще не грузить мод — не регистрировалась ни
одна команда. Теперь мод загружается и все функции работают без WorldEdit; он нужен только для
`/rg claim` и `/rg redefine` (читают выделение). Если WorldEdit нет, эти две команды отвечают
понятным сообщением (`msg.selection.no-worldedit`, EN+RU) вместо тихого отказа. Добавлен
`WorldEditAdapter.isAvailable()`.

**Чистка репозитория:** удалён каталог `tests/` (JVM-наборы и `coverage.sh`) и разделы про
тесты в `BUILD.md`/`BUILD_RU.md`. README (EN+RU) и BUILD (EN+RU) обновлены — WorldEdit указан
как опциональный. В репозитории остаётся только сам мод.

## build 13 — 2026-06-13

Закрыты последние два грифинг-вектора из аудита b12: грифинг блоков мобами и тросовые крюки.

**Новый флаг `mob-grief`** (state, по умолчанию allow): запрещает мобам менять блоки на клейме —
эндермен берёт/ставит блок, овца ест траву, зомби ломает дверь, силверфиш, фермер-житель и т.д.
Ваниль проводит всё это через `EntityMobGriefingEvent`; обработчик `onMobGriefing` вызывает
`setCanGrief(false)`, если `mob-grief` в позиции моба = deny. Не трогает мировое игровое правило
`mobGriefing` — защита точечная, по региону. Всего флагов: **84**.

**Tripwire (тросовый крюк)** — новый `TripWireBlockMixin` на `TripWireBlock.entityInside`
(сигнатура сверена по декомпилированному MC). Тросовый крюк в защищённом регионе не срабатывает
от чужих игроков, мобов и брошенных предметов — тот же класс грифа, что и нажимные плиты (для
которых миксин уже был). Член региона (build-access `use`) активирует трос как обычно. Быстрый
путь по `hasAnyAt`, `require=0` (при ремапе просто не применится, без краша). Редстоун-выход
троса дополнительно регулируется флагом `redstone`.

**Тесты:** новый флаг и поведение автоматически подхвачены наборами (резолюция, контракт,
round-trip, страж покрытия) — счётчики выросли, все зелёные (~1240 JVM-проверок). `mob-grief`
проходит проверку enforcement-покрытия.

**Документация:** FLAGS.md/FLAGS_RU.md перегенерированы (84 флага); README — пункт о контроле
грифа мобами и ловушек (EN+RU); CHANGELOG.

проверка подтвердила, что база чистая. Сборка идентична build 1 по коду; отличается

- Сериализация флагов: StateFlag.parse locale-safe (Locale.ROOT — защита от
  турецкого I→ı), null=unset обрабатывается корректно.

- Vec3 — record (безопасный equals/hashCode для Map/Set).

## build 1 — 2026-06-01

Первая сборка с системой нумерации (раньше все архивы назывались одинаково).
Эта сборка фиксирует состояние после общего аудита кода.

**Аудит кода и оптимизация:**
- `onFireSpread` (самый горячий обработчик — срабатывает на каждый neighbor-notify):
  объединены два прохода по сторонам блока в один; убрана аллокация `HashSet`
  (`regionIdSet`) на каждую сторону — теперь прямой обход `getApplicable`.
  World-выключатели (`preventFireSpread`/`preventLavaFire`) резолвятся один раз до цикла.
- Удалён мёртвый метод `regionIdSet` из `WorldEventHandler` (больше не используется).
- `WGConfig` (TOML): парсер `TomlFormat.createParser().parse()` теперь читается
  напрямую в `CommentedConfig`, без лишнего промежуточного `putAll`.

**Проверено (без изменений — уже корректно):**
- Баланс скобок, отсутствие неиспользуемых импортов, отсутствие утечек ресурсов
  (все `Files.list/lines/newBufferedReader` в try-with-resources).
- Потокобезопасность `ViolationLog` (ArrayBlockingQueue + daemon-поток + volatile-флаги).
- Все 12 mixin'ов: null-проверка `WorldGuardNeo.get()`, fast-path, fail-open.
- `RegionContainer.get(Level)` — горячий путь без аллокаций (IdentityHashMap).
- Нет stream()/collect в горячих обработчиках.

**Состояние мода на момент сборки:**
- 60 Java-файлов, 12 mixin'ов, 83 флага (80 рабочих, 3 задекларированы).
- Конфиг в TOML с комментариями (`config/worldguardneo/config.toml`), регионы в JSON.
- Защита смежных регионов A↔B: поршни (дроп при нарушении), жидкость/огонь
  (boundary), рост деревьев, диспенсер (`dispenser-output` + boundary).
