## build 15 — 2026-06-14

Полный аудит кода (движок регионов, слушатели событий, слой хранилища, права, флаги, миксины).
Исправлены реальные дефекты корректности, безопасности и потери данных; пара точечных
оптимизаций. Чисто высококачественный код — большинство подсистем прошли аудит без замечаний.

**Потеря данных (хранилище):**
- *Битый JSON больше не уничтожает регионы.* Если файл региона повреждён/обрезан, загрузка
  раньше оставляла мир ПУСТЫМ, а следующее автосохранение перезаписывало целый файл пустыми
  данными — необратимая потеря. Теперь повреждённый файл переименовывается в
  `<имя>.corrupt-<метка>` (карантин), сохранение пишет свежий файл, а оригинал можно
  восстановить из карантина или бэкапа (`JsonRegionStorage`).
- *Долговечность записи.* Перед атомарным переименованием данные теперь сбрасываются на диск
  через `FileChannel.force(true)` — потеря питания больше не оставляет обрезанный/пустой файл.
  Кодировка чтения/записи явно зафиксирована как UTF-8.
- *MySQL.* В `CREATE TABLE` добавлен `DEFAULT CHARSET=utf8mb4`, чтобы не-ASCII (id регионов,
  приветствия, ключи модовых измерений) корректно сохранялись на серверах с latin1 по умолчанию.

**Корректность защиты (слушатели):**
- `onExplosion` теперь уважает мировой выключатель `useRegions=false` — раньше взрывы
  фильтровались флагами регионов даже в мире с отключённым WGN.
- `onExpDrop`: опыт **игроков** при смерти больше не обнуляется флагом `exp-drops` (он для
  мобов) — этим владеет исключительно обработчик `keep-xp`, конфликт устранён.
- `onRightClickBlock`: если определить «это контейнер?» не удалось (гонка при выгрузке чанка),
  взаимодействие теперь блокируется (fail-safe), а не пропускается молча — закрывает потенциальную
  утечку доступа к сундукам.

**Движок регионов:**
- `SpatialIndex`: регион с экстремальными координатами (на краю мира) больше не выпадает из
  индекса молча (его флаги переставали действовать) — он помещается в fallback-набор oversized.

**Оптимизации:**
- `onProjectileImpactEntity` сразу выходит, если событие уже отменено.
- `onTreeGrow` пропускает скан 7×7 в мирах без регионов (частый случай для Ада/Энда).

Все изменения собираются (`./gradlew build`). Ложноположительные находки аудита
(`dispenser-output` уже enforced через миксин и т.п.) проверены и отклонены.

**Ручная доглубокая проверка (миксины, команды, права, util, жизненный цикл, интеграции, флаги):**
подсистемы, не дошедшие до отчёта на этапе авто-аудита, прочитаны вручную целиком. Реальных
багов не найдено — код качественный (миксины fail-open с глобальным `require=0`; команды строго
проверяют права: владелец/bypass/others, пер-флаговые узлы, узел `-g`, защита глобального региона,
лимиты claim переприменяются в redefine; `UuidResolver` не ходит в сеть; `ViolationLog`
асинхронный с graceful-shutdown). Поправлена устаревшая документация после перевода WorldEdit в
опциональные (`WorldEditAdapter` больше не называет WE «required» и не ссылается на удалённый
`/rg select`; комментарий в `OpResolver` о уровне bypass приведён в соответствие с кодом, level 5).

**Несколько отдельных сквозных глубоких проверок (вручную):**
1. *Полнота enforcement флагов* — все 82/84 флага реально применяются; 2 (`allowed-enchants`,
   `receive-chat`) объявлены-но-не-применяются и документированы как таковые (EN+RU). ✅
2. *Усечение координат* — нигде нет `(int)`-каста на double-координаты; везде `Math.floor`
   или raw-double с floor внутри индекса (нет off-by-one на отрицательных координатах). ✅
3. *Семантика границ (cross-border)* — поршни/раздатчики используют явный `== ALLOW` (opt-in),
   огонь/лава/вода блокируют любой переход границы; опасного `!= DENY` (unset=allow) нет. ✅
4. *Согласованность узлов прав* — все 28 узлов совпадают между кодом, `OpResolver` и
   PERMISSIONS.md; пер-флаговые узлы `worldguardneo.flag.<name>` (опечаток нет). ✅
5. *Покрытие мирового выключателя* — **найден и исправлен реальный пробой:** цепочка
   смерти/респауна (`onPlayerDeath` снимок keep-inventory/keep-xp + позиция для spawn-loc)
   не уважала `useRegions=false`, поэтому в мире с отключённым WGN флаги keep-inventory/
   keep-xp/spawn всё равно срабатывали. Гейт добавлен в единую точку снимка (downstream
   читает снимок). ✅
6. *Согласованность bypass* — каждая проверка `region.bypass` идёт по действующему игроку,
   в правильную сторону, узел только явный (OP-5); инверсий/эксплойтов нет. ✅

## build 14 — 2026-06-14

Критическое исправление видимости команд и переход WorldEdit в опциональные зависимости —
плюс чистка репозитория до «только мод».

**Исправлена видимость `/rg` и `/region`** (`LuckPermsResolver.has`). Раньше при результате
`UNDEFINED` (узел не упомянут в LuckPerms — обычный случай, когда правами управляют через
группы) метод жёстко возвращал deny и **игнорировал уровень OP**. Brigadier удаляет из дерева
команд игрока любую ветку, чей `requires` ложный, поэтому всё дерево `/rg` пропадало —
команды выглядели как «несуществующие». Теперь модель стандартная «LuckPerms переопределяет,
OP — это дефолт»: `TRUE` → allow, `FALSE` → deny (LP может отозвать доступ у OP),
`UNDEFINED` → откат на OP-уровень узла (`OpResolver`). `region.bypass` остаётся OP-5 и по-прежнему
никогда не выдаётся одним лишь OP.

**WorldEdit теперь опционален** (`neoforge.mods.toml`: `type="required"` → `type="optional"`).
Раньше отсутствие WorldEdit заставляло NeoForge вообще не грузить мод — не регистрировалась ни
одна команда. Теперь мод загружается и все функции работают без WorldEdit; он нужен только для
`/rg claim` и `/rg redefine` (читают выделение). Если WorldEdit нет, эти две команды отвечают
понятным сообщением (`msg.selection.no-worldedit`, EN+RU) вместо тихого отказа. Добавлен
`WorldEditAdapter.isAvailable()`.

**Чистка репозитория:** удалён каталог `tests/` (JVM-наборы и `coverage.sh`) и разделы про
тесты в `BUILD.md`/`BUILD_RU.md`. README (EN+RU) и BUILD (EN+RU) обновлены — WorldEdit указан
как опциональный. В репозитории остаётся только сам мод.

## build 13 — 2026-06-13

Закрыты последние два грифинг-вектора из аудита b12: грифинг блоков мобами и тросовые крюки.

**Новый флаг `mob-grief`** (state, по умолчанию allow): запрещает мобам менять блоки на клейме —
эндермен берёт/ставит блок, овца ест траву, зомби ломает дверь, силверфиш, фермер-житель и т.д.
Ваниль проводит всё это через `EntityMobGriefingEvent`; обработчик `onMobGriefing` вызывает
`setCanGrief(false)`, если `mob-grief` в позиции моба = deny. Не трогает мировое игровое правило
`mobGriefing` — защита точечная, по региону. Всего флагов: **84**.

**Tripwire (тросовый крюк)** — новый `TripWireBlockMixin` на `TripWireBlock.entityInside`
(сигнатура сверена по декомпилированному MC). Тросовый крюк в защищённом регионе не срабатывает
от чужих игроков, мобов и брошенных предметов — тот же класс грифа, что и нажимные плиты (для
которых миксин уже был). Член региона (build-access `use`) активирует трос как обычно. Быстрый
путь по `hasAnyAt`, `require=0` (при ремапе просто не применится, без краша). Редстоун-выход
троса дополнительно регулируется флагом `redstone`.

**Тесты:** новый флаг и поведение автоматически подхвачены наборами (резолюция, контракт,
round-trip, страж покрытия) — счётчики выросли, все зелёные (~1240 JVM-проверок). `mob-grief`
проходит проверку enforcement-покрытия.

**Документация:** FLAGS.md/FLAGS_RU.md перегенерированы (84 флага); README — пункт о контроле
грифа мобами и ловушек (EN+RU); CHANGELOG.

WorldEdit в classpath. Это устраняет привязку к координатам/версии артефакта.
