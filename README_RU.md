WorldGuardNeo приносит защиту территорий в стиле WorldGuard на NeoForge. Вы выделяете область через **WorldEdit**, регистрируете её как регион и управляете тем, что внутри можно делать, с помощью большого набора флагов — строительство, PvP, спавн мобов, огонь, взрывы, редстоун, жидкости и многое другое. Мод также защищает **между смежными регионами**, так что сосед не сможет гриферить вас через общую границу. Всё настраивается в TOML-файле с комментариями и применяется на лету через `/rg reload`.

## Возможности

- **Регионы по выделению WorldEdit** — выделяете через `//wand` (куб или полигон) и регистрируете. Отдельный wand учить не нужно.
- **80+ флагов** — build, block-break/place, interact, use, chest-access, pvp, спавн и урон мобов, взрывы (tnt/creeper/прочие), распространение огня, поджог лавой, молнии, редстоун, поршни, выпуск из диспенсеров, жидкости, рост, вход/выход, приветствия, блокировки режима/времени/погоды, сохранение инвентаря/опыта и другое.
- **На регион и на мир** — флаги на каждый регион; общемировые переключатели на измерение через файлы-переопределения.
- **Модель участников** — владельцы и участники; build-флаги их учитывают (WorldGuard «приватно по умолчанию»).
- **Родители и приоритет** — регион наследует флаги от родителя; перекрывающиеся регионы разрешаются по приоритету (DENY важнее ALLOW).
- **Защита от грифа на границе** — распространение лавы/воды/огня, поршни, диспенсеры и рост деревьев блокируются при попытке пересечь границу в чужой регион.
- **Сменные хранилища** — `json` (по умолчанию), `sqlite`, `h2` или `mysql`. Любой БД-бэкенд откатывается на JSON, если драйвера нет.
- **Интеграция с LuckPerms** — права через LuckPerms при наличии (иначе откат на уровни OP), плюс лимиты регионов на группу.
- **Интеграция с веб-картами** — регионы отображаются на **BlueMap** и **squaremap** при наличии.
- **Автоматические бэкапы** — по расписанию, ротация, gzip-сжатие.
- **Локализация** — английский и русский «из коробки»; добавьте `<tag>.json` для других языков.

## Требования

**Обязательно**

- Minecraft **1.21.1**
- NeoForge **21.1.x**
- **WorldEdit** для NeoForge **7.3+** — регионы создаются по его выделению

**Опционально**

| Зависимость | Что включает |
| --- | --- |
| **LuckPerms** 5.4+ | Узлы прав + лимиты регионов на группу (иначе уровни OP) |
| **BlueMap** | Отображение регионов на 3D веб-карте |
| **squaremap** | Отображение регионов на 2D веб-карте |
| **sqlite-jdbc** jar | `storage-format = "sqlite"` |
| **H2** jar | `storage-format = "h2"` (H2 уже поставляет LuckPerms) |
| **mysql-connector-j** jar | `storage-format = "mysql"` |
| **JDK 21** | Только для сборки из исходников |

Мод **только серверный**. Ванильные клиенты подключаются как обычно, ставить что-либо на клиент не нужно. WorldEdit рисует рамку выделения (через WECUI) для клиентов, у которых он есть. Любой БД-бэкенд без драйвера автоматически откатывается на JSON, так что отсутствие опционального jar не останавливает сервер.

## Установка

1. Положите `.jar` WorldGuardNeo **и WorldEdit** в папку `mods/` сервера.
2. *(Опционально)* добавьте LuckPerms, BlueMap или squaremap.
3. Запустите сервер один раз, чтобы создался `config/worldguardneo/config.toml`.
4. При необходимости измените настройки и выполните `/rg reload`.

## Быстрый старт

```
//wand                 (WorldEdit) ЛКМ/ПКМ по двум углам
/rg claim myregion     зарегистрировать выделение как регион "myregion"
/rg flag myregion pvp deny
/rg addmember myregion Steve
```

`/rg info myregion` также подсвечивает контур региона, загружая его границы в ваше выделение WorldEdit (отрисовывается WECUI), чтобы `/rg redefine myregion` мог изменить его размер после нового выделения.

## Настройка

Все параметры — в `config/worldguardneo/config.toml`, TOML-файле с комментарием к каждому ключу. Переопределения на мир — в `config/worldguardneo/worlds/<измерение>.toml` (`:` заменяется на `_`), наследуют глобальную секцию `[defaults]`. Сами регионы хранятся отдельно в `config/worldguardneo/regions/`.

### Хранилище

```toml
# json | sqlite | h2 | mysql  (требует перезапуска сервера)
storage-format = "json"
```

- **json** — один файл на мир, без зависимостей (по умолчанию).
- **sqlite** — встроенный `regions.sqlite`; нужен sqlite-jdbc.
- **h2** — встроенный `regions_h2`; нужен H2 (его поставляет LuckPerms).
- **mysql** — внешний сервер; настройте `[mysql]` ниже и добавьте `mysql-connector-j`.

Любой БД-бэкенд без драйвера автоматически откатывается на `json`.

```toml
[mysql]
host = "localhost"
port = 3306
database = "worldguardneo"
user = "root"
password = ""
use-ssl = false
table = "world_regions"             # измените, чтобы делить одну БД между серверами
connection-timeout-seconds = 10
properties = ["serverTimezone=UTC"] # доп. параметры JDBC "ключ=значение" к URL
```

Бэкенд MySQL принимает драйвер **Connector/J** *или* **MariaDB** и открывает соединение через драйвер напрямую (минуя `DriverManager`), поэтому работает даже когда jar драйвера загружен другим class-loader'ом — это и была причина прежних «DB init failed → откат на JSON». Тот же путь используется для SQLite и H2.

### Лимиты регионов на группу (LuckPerms)

```toml
[group-region-limits]
default = 7
vip = 15
premium = 30
```

Игрок получает **наибольший** лимит среди своих групп LuckPerms. Без LuckPerms ко всем применяется `max-regions-per-player`.

### Бэкапы

```toml
[backup]
enabled = true
interval-minutes = 60
retain-count = 10
compress = true
```

### Значения по умолчанию на мир

```toml
[defaults]
use-regions = true
disable-explosions-around-regions = true
prevent-fire-spread = false
prevent-lava-fire = false
# ...полный список — в файле, каждый ключ с комментарием
```

### Авто-флаги и вертикальная защита (на мир)

Каждый новый регион в мире может настраиваться автоматически. Это задаётся на мир (в `[defaults]` или в переопределении `worlds/<измерение>.toml`):

```toml
[defaults]
# Флаги, применяемые к каждому новому региону, в формате "флаг=значение".
auto-flags = ["pvp=deny", "mob-spawning=deny", "creeper-explosion=deny"]

# Авто-расширение новых регионов по вертикали, чтобы никто не мог подкопаться снизу
# или зайти по мосту сверху: "none" | "full" | "fixed".
vertical-expansion = "full"
vertical-expand-down = 0   # для режима "fixed"
vertical-expand-up = 0     # для режима "fixed"
```

- **`vertical-expansion = "full"`** растягивает регион от бедрока до потолка стройки — регион, созданный на поверхности, защищён и под землёй, и в небе. Рекомендуемая анти-гриф настройка.
- **`"fixed"`** расширяет на заданное число блоков вниз/вверх от выделения.
- Горизонтальные лимиты размера проверяются на **исходном** выделении игрока, так что вертикальное расширение не расходует его лимит площади.
- Авто-флаги и расширение применяются только к **новым** регионам; существующие не затрагиваются.

## Команды

Работают обе формы — `/region` и `/rg`. Идентификатор региона — одно слово.

| Команда | Описание |
| --- | --- |
| `/rg claim <id>` | Зарегистрировать выделение WorldEdit как новый регион |
| `/rg redefine <id>` | Изменить размер региона под текущее выделение |
| `/rg remove <id>` | Удалить регион |
| `/rg info [id]` | Показать тип, размер, границы, владельцев, участников, флаги — и подсветить контур |
| `/rg list [игрок]` | Список регионов (опц. для игрока) |
| `/rg lists [радиус]` | Список регионов рядом с вами (радиус по умолчанию 50) |
| `/rg flag <id> <флаг> [-g группа] [значение]` | Задать/снять флаг (без значения — снять) |
| `/rg flags` | Список всех доступных флагов с подсказкой формата и описанием |
| `/rg priority <id> <n>` | Задать приоритет региона (только владелец или админ) |
| `/rg setparent <id> [родитель]` | Задать/снять родителя региона (только владелец или админ) |
| `/rg addmember/removemember <id> <игрок>` | Управление участниками |
| `/rg addowner/removeowner <id> <игрок>` | Управление владельцами |
| `/rg teleport <id>` | Телепорт к точке флага `teleport` региона |
| `/rg backup [метка]` | Создать бэкап регионов сейчас (опц. суффикс имени) |
| `/rg backup list` | Список существующих бэкапов, новые сверху |
| `/rg save` | Сбросить регионы в хранилище |
| `/rg reload` | Перезагрузить конфиг и язык |
| `/rg debug` | Диагностика: статистика индекса, активные интеграции |

## Права

При установленном LuckPerms он — единственный источник прав (уровни OP игнорируются после загрузки игрока). Без него каждый узел маппится на уровень OP. Дефолты **щадящие для базового использования**: создание, просмотр и удаление своих регионов — OP 0 (все), а управление и админ-действия требуют более высоких уровней.

| Узел | По умолчанию | Назначение |
| --- | --- | --- |
| `worldguardneo.region.claim` | OP 0 | Регистрация регионов |
| `worldguardneo.region.delete` | OP 0 | Удаление своих регионов |
| `worldguardneo.region.info` / `.list` | OP 0 | Просмотр / список своих регионов |
| `worldguardneo.region.teleport` | OP 0 | Телепорт к регионам |
| `worldguardneo.selection.use` | OP 0 | Получение кастомного `//wand` |
| `worldguardneo.region.redefine` | OP 2 | Изменение размера |
| `worldguardneo.region.flag` | OP 2 | Установка флагов |
| `worldguardneo.region.addmember` / `addowner` | OP 2 | Управление участниками / владельцами |
| `worldguardneo.region.delete.others` | OP 3 | Удаление чужих регионов |
| `worldguardneo.region.bypass` | **OP 5** | Обход защиты (только LuckPerms — по OP не выдаётся) |
| `worldguardneo.backup` / `reload` | OP 4 | Ручной бэкап / перезагрузка |

**Полный список** всех узлов, точные уровни OP и примеры LuckPerms — в [PERMISSIONS_RU.md](PERMISSIONS_RU.md).

## Флаги

WorldGuardNeo поставляет 80+ флагов. Каждый документирован в игре — выполните `/rg flags`, чтобы увидеть полный список с подсказками формата и описаниями, а `/rg info <id>` покажет, что установлено на регионе. Build-флаги (build, block-break, block-place, interact, use, chest-access) следуют модели участников; остальные — средовые state-флаги (allow/deny) или флаги-значения (приветствия, блокировки, лимиты).

## Сборка

Нужен JDK 21. Из папки проекта:

```
./gradlew clean build
```

Готовый `.jar` появится в `build/libs/`. WorldEdit **не** является зависимостью времени компиляции — мод обращается к нему только через рефлексию, поэтому собирается без WorldEdit в classpath. WorldEdit обязателен в **рантайме** (объявлен в `neoforge.mods.toml`).
